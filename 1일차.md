## skill.md
 - 모듈화
 - 반복&공통등에 적용
 - 코드 리뷰를 일관되게도 사용가능
 - name, description 필수 옵션
 - /context시에 skill 사용량을 확인 가능
 - 알아서 에이전트가 활용 (global)
 - 명시적으로만 사용 가능 (local, '/')
   - 커스텀 명령어로 만들 수 있다.
 - Sub Agent 별로 나누어 사용
   - FE, BE 각 스킬이 필요한 
   - Skill은 하나의 일의 덩어리
 - https://code.claude.com/docs/ko/skills
 - 구현 예시)
   - FE, BE, kernal, DB 테스트 코드를 작성하는 스킬을 만들어보자
   - FE 테스트 하는 스킬 만들어보자.
 - 참조 형태를 권장한다. (reference의 경우 상세한 내용을 활용)
   
## 서브 에이전트
 - 클로드는 메인 에이전트라고 한다. 모든 작업은 이 아이가 처리함.
 - 하나의 에이전트에서는 다수의 작업 수행시 Context-window의 한계에 도달할 수 있다.
  - context-window가 많을 시 할루시네이션, 성능, 비용이 증가한다.
 - FE, BE, DB 등으로 구분하여 관심사를 분리해야함.
 - 메인 에이전트에 서브 에이전트 (FE, BE, DB) 등으로 관련된 내용만 처리시 Context-window를 효율적
 - 서브 에이전트
   - skills와 같은 방식으로 작성
   - tool (mcp)를 활용한다. (크롤링, 웹검색, 피그마 API)
   - skills
   - 단점: Content를 공유하지 못한다.
     - 극복방안: 로깅을 작성한다.
   - cli에서 /agents로 에이전트를 만들 수 있다.
     - personal을 전역임
   - 최소 500라인, 가능한 200~300라인 유지하는게 좋음 (프롬프트도 유지보수 대상임)
   - https://code.claude.com/docs/ko/sub-agents
